# Issue tracker

Issues are tracked as local markdown files under `.scratch/<feature>/`.

## Commands

- **Create an issue**: Write a markdown file to `.scratch/<feature>/`.
- **List issues**: `ls .scratch/<feature>/`
- **Read an issue**: Read the markdown file.
- **Update an issue**: Edit the markdown file.

## PRs as a request surface

PRs are not tracked as issues in this repo (flag: off).
